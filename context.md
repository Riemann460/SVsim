# 공유 컨텍스트 (context.md)

## 1. 비즈니스 도메인 용어 정의
* **추종자 (Follower)**: 전장에 소환되어 전투를 수행하고 진화/초진화할 수 있는 카드 타입.
* **주문 (Spell)**: 플레이 시 효과를 발동하고 즉시 묘지로 이동하는 일회성 카드 타입.
* **마법진 (Amulet)**: 전장에 소환되어 지속적인 효과를 주거나 카운트다운 후 효과를 발동하는 카드 타입.
* **리더 (Leader)**: 플레이어 자신 혹은 상대방을 나타내며, 체력이 0 이하가 되면 게임에서 패배함.
* **진화 (Evolution)**: EP를 소모하여 아군 추종자를 강화하고 소환된 턴에 추종자를 공격할 수 있게 하는 메커니즘.
* **초진화 (Super Evolution)**: SEP를 소모하여 아군 추종자를 한 단계 더 높은 수준으로 강화하는 특수 진화 메커니즘.
* **증강 (Enhance)**: 특정 카드를 플레이할 때 추가 PP를 소모하여 더욱 강력한 효과를 발동하는 메커니즘.
* **주문 증폭 (Spellboost)**: 주문 카드를 플레이할 때마다 패에 있는 특정 카드의 코스트를 감소시키거나 효과를 강화하는 메커니즘.
* **엑스트라 PP (Extra PP)**: 기본 PP 외에 추가적으로 사용할 수 있는 특수 자원.

## 2. 아키텍처 결정 사항 (ADR)
* **ADR-001: 이벤트 기반 아키텍처 (Event-Driven Architecture)**
  * **결정**: `EventManager`와 `EventType`을 사용하여 게임 내 발생하는 각종 상태 변화(전장 소환, 카드 플레이, 파괴 등)를 이벤트로 발행하고, `EffectProcessor`에서 이를 구독하여 처리.
  * **이유**: 복잡한 카드 간 상호작용과 트리거 효과(출격, 유언, 진화시 등)를 느슨한 결합으로 유연하게 처리하기 위함.
* **ADR-002: Tkinter 기반의 그래픽 사용자 인터페이스 (GUI)**
  * **결정**: 플레이어의 행동 입력 및 게임 상태 모니터링을 위해 `tkinter` 라이브러리를 활용한 간단한 GUI 구현.
  * **이유**: 별도의 무거운 웹 프레임워크나 게임 엔진 없이 파이썬 내장 라이브러리로 신속한 로컬 테스트 환경을 구축하기 위함.
* **ADR-003: 객체 지향 도메인 모델**
  * **결정**: `Player`, `Card`, `Deck`, `Hand`, `Field`, `Graveyard` 등 게임 도메인 핵심 요소를 각각의 클래스로 독립적으로 캡슐화.
  * **이유**: 실제 카드 게임의 물리적/개념적 영역을 투명하게 모델링하여 코드의 가독성과 확장성을 확보하기 위함.
* **ADR-004: 데이터 구동형 카드 디자인 (Data-Driven Card Design)**
  * **결정**: 카드의 스탯과 효과 구조를 JSON 데이터로 정의하고, 게임 시작 시 `card_data.py`를 통해 메모리에 로드하여 인스턴스화.
  * **이유**: 코어 엔진 수정 없이 JSON 데이터 추가/변경만으로 새로운 카드를 손쉽게 확장하기 위함.
