# 디버깅 작업 흐름 스킬

## 1. 개요
퍼징 테스트 등에서 런타임 결함이 발견되었을 때, 임시 방어 가드 코드를 작성하는 대신 문제의 근본적인 원인을 제거하여 시스템의 안정성을 확보하는 개발 지침입니다.

## 2. 핵심 원칙
- **임시 방어 코드 금지**
  속성 누락이나 TypeError 등을 예방하기 위한 getattr 남용이나 __getattr__을 통한 None 반환 처리 등 임시 가드 코드는 작성하지 않습니다.
- **근본 원인 제거**
  효과가 누락되어 파싱된 경우 파이프라인의 정규식 패턴 분석 결함을 해결하는 것과 같이 데이터 파싱이나 생성 단계에서 원인을 추적하여 완전히 해결합니다.
- **사용자 감독 하 진행**
  원인 규명 후 본 코드를 수정하기 전 반드시 사용자에게 발생한 오류와 수정 계획에 대해 설명하고, 승인을 받으면 진행합니다. 
- **시나리오 기반 재현 테스트 준수**
  에러가 발생했을 때의 스냅샷 시나리오를 기반으로 시나리오 단위 테스트를 작성하여 재현 및 검증을 수행하며, 작성한 시나리오 테스트 코드는 유지합니다.

## 3. 세부 단계별 지침
1. **문제 발생 분석**
   오류 로그의 트레이스백과 스냅샷을 분석하여 어떤 카드의 효과를 처리할 때 오류가 발생했는지 확인합니다.
2. **원인 추적**
   카드의 원본 텍스트와 파서 정규식을 비교하여 파싱 결과에서 누락된 속성이나 값의 발생 원인을 진단합니다.
3. **상황 보고**
   사용자에게 스냅샷과 추적한 원인의 내용 및 수정 방향을 제시하고, 확인 받습니다.
4. **시나리오 재현 테스트 작성**
   에러가 발생했을 때의 스냅샷 상황을 기반으로 시나리오 단위 테스트를 작성하여 실패 및 성공 여부를 검증하며, 이 테스트 코드는 보존합니다.
5. **본 코드 구현 및 승인**
   파서 패턴 또는 게임 엔진의 버그 원인을 근본적으로 제거하며, 수정 전에 사용자의 동의를 구합니다.
6. **검증 및 커밋**
   시나리오 재현 테스트가 정상 통과하는지 검증하고, 검증 완료 후 최소 의미 단위로 Git 커밋을 진행합니다.
